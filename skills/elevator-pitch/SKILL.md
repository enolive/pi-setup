# Elevator Pitch Skill

Use this skill when the user needs a short spoken pitch for a busy executive.

## Core principle

An elevator pitch is not a slide deck, research note, or architecture summary.
It should be short enough to say aloud in about 30-60 seconds.

## Structure

Use 3 short paragraphs:

1. **Problem** — what pain exists and why it matters.
2. **Current gap** — why the current state or existing alternatives are not enough.
3. **Proposal/value** — what investment is requested and what outcome it creates.

## Style

- Plain language.
- No implementation details unless essential.
- No long bullet lists.
- No more than one or two product/technical names.
- Focus on value, trust, risk, time saved, or decision confidence.
- Write it so someone could literally say it in an elevator.

## Example pattern

```md
AI coding agents are starting to work unattended, but humans still miss the important moments: when the agent finishes, fails, or needs attention.

We already have a prototype, but it is silent when it breaks and has no tests, so we cannot trust it enough to publish.

The proposal is a small investment: turn the prototype into a minimal tested adapter with opt-in diagnostics. It keeps policy in the right place, keeps the integration tiny, and makes long-running sessions easier to trust.
```

## What to avoid

- Slide-style headings.
- Dense lists.
- Market research detail.
- Technical architecture tours.

If the user wants more detail, write a separate management slide deck or research document and link to it.
